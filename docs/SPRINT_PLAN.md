# Sprint Plan — Bankruptcy AI Platform

| Фаза | Срок | Цель | Результат |
|------|------|------|-----------|

| Sprint 2 | Недели 4-5 | AI-квалификация | Чат-бот + скоринг + лендинг |

## Sprint 0: Инфраструктура (Неделя 1)

### Задачи
- [x] Создать монорепо и структуру проекта
- [x] Спроектировать схему БД
- [x] Настроить Docker Compose (PostgreSQL + Redis + MinIO)
- [x] Создать скаффолдинг FastAPI бэкенда
- [x] Создать скаффолдинг AI Core
- [ ] Настроить GitHub репо + branch protection
- [ ] Настроить CI (GitHub Actions): lint, tests, build
- [ ] Настроить Alembic для миграций
- [ ] Деплой dev-окружения (Yandex Cloud / VPS)
- [ ] Настроить мониторинг (Sentry + Grafana)

### Definition of Done
- `docker compose up` поднимает всё
- Миграции применяются
- Health check отвечает 200
- CI проходит на push

## Sprint 1: CRM-ядро (Недели 2-3)

### Backend
- [ ] Auth: JWT login/register, password hashing (bcrypt)
- [ ] RBAC middleware: проверка ролей на эндпоинтах
- [ ] CRUD клиентов (создание, поиск, редактирование)
- [ ] CRUD дел (создание, воронка статусов, назначение юриста)
- [ ] CRUD кредиторов (привязка к делу, пересчёт суммы долга)
- [ ] API таймлайна событий по делу
- [ ] API дедлайнов (создание, контроль, напоминания)
- [ ] API платежей (счёт, оплата, статус)
- [ ] Пагинация и фильтрация на всех листингах
- [ ] Аудит-лог: логирование всех изменений
- [ ] Unit-тесты на основные эндпоинты

### Frontend (Next.js)
- [ ] Настроить Next.js 14 + TypeScript + Tailwind
- [ ] Layout: сайдбар + хедер + контентная область
- [ ] Страница логина
- [ ] Список дел (таблица с фильтрами по статусу, юристу)
- [ ] Карточка дела (базовая: инфо + статус + кредиторы)
- [ ] Список клиентов
- [ ] Форма создания клиента + дела
- [ ] Настроить API-клиент (fetch wrapper с токеном)

### Definition of Done
- Можно создать клиента → создать дело → добавить кредиторов → сменить статус
- Все действия логируются в аудит
- Роли работают (юрист видит свои дела, админ — все)

## Sprint 2: AI-квалификация (Недели 4-5)

### AI Core
- [ ] Агент квалификации: правила + LLM scoring
- [ ] Rule-based pre-screening (без LLM, быстрый фильтр)
- [ ] Интеграция с Claude API
- [ ] Сохранение результатов скоринга в карточку дела
- [ ] API для чат-бота (stateful conversation)
- [ ] Логирование AI-задач (input/output/confidence/cost)

### Frontend — Лендинг
- [ ] Лендинг: hero + преимущества + форма + FAQ
- [ ] Виджет чат-бота квалификации
- [ ] Логика: вопрос-ответ → скоринг → результат → запись на консультацию
- [ ] UTM-трекинг (сквозная аналитика)
- [ ] Мобильная адаптация лендинга

### Backend
- [ ] Эндпоинт /ai/qualify → вызов AI Core
- [ ] Эндпоинт создания лида из чат-бота (клиент + дело со статусом lead)
- [ ] Webhook для записи на консультацию

### Definition of Done
- Человек заходит на сайт → чат-бот задаёт вопросы → получает оценку
- Лид создаётся в CRM автоматически
- Юрист видит дело со скорингом AI

## Sprint 3: Документы (Недели 6-7)

### AI Core
- [ ] OCR-пайплайн: загрузка → распознавание → извлечение данных
- [ ] Классификатор типа документа (паспорт, 2-НДФЛ, кредитный договор)
- [ ] Извлечение структурированных данных из документов
- [ ] Кросс-валидация между документами

### Backend
- [ ] Загрузка файлов в MinIO/S3
- [ ] Очередь задач на OCR (Redis + worker)
- [ ] API чек-листа документов по делу
- [ ] Автоматические запросы недостающих документов клиенту
- [ ] Версионирование документов

### Frontend
- [ ] Страница документов дела (загрузка, просмотр, статусы)
- [ ] Визуальный чек-лист (галочки + прогресс-бар)
- [ ] Превью загруженных документов
- [ ] Уведомления о недостающих документах

### Definition of Done
- Клиент загружает фото паспорта → AI определяет тип → извлекает ФИО, серию/номер
- Чек-лист показывает прогресс сбора
- Юрист видит извлечённые данные и может поправить

### Frontend
- [ ] Дашборд юриста: мои дела, дедлайны, задачи
- [ ] Канбан-доска дел (колонки по статусам, drag-and-drop)
- [ ] Календарь дедлайнов и заседаний
- [ ] Детальная карточка дела: все вкладки (инфо, кредиторы, документы, таймлайн, платежи)
- [ ] Форма редактирования дела
- [ ] Нотификации (in-app)

### Backend
- [ ] API нотификаций
- [ ] Cron: проверка дедлайнов, генерация напоминаний
- [ ] API статистики по юристу (количество дел, сроки, загрузка)

### Analytics
- [ ] Воронка конверсий (view на SQL уже есть)
- [ ] Загрузка юристов
- [ ] Средние сроки по этапам
- [ ] Юнит-экономика по делам

### Definition of Done
- Юрист видит все свои дела, ближайшие дедлайны, прогресс по документам
- Дашборд с ключевыми метриками
- Нотификации о критичных дедлайнах

## Sprint 5: Личный кабинет клиента (Недели 10-11)

### Frontend
- [ ] Авторизация клиента (по телефону + SMS код)
- [ ] Главная: статус дела (визуальный прогресс-бар по этапам)
- [ ] Чат с AI-ботом (типовые вопросы: «когда заседание?», «какие документы нужны?»)
- [ ] Загрузка документов из ЛК
- [ ] История событий по делу (видимые для клиента)
- [ ] Уведомления (push / email / SMS)

### AI Core
- [ ] Клиентский бот-ассистент: ответы на вопросы по делу
- [ ] Генерация еженедельного апдейта для клиента

### Backend
- [ ] Auth для клиентов (отдельный от сотрудников)
- [ ] API клиентского ЛК (ограниченный доступ только к своему делу)
- [ ] Отправка уведомлений (email + SMS интеграция)

### Definition of Done
- Клиент логинится → видит статус дела → может задать вопрос боту
- Автоматические уведомления о событиях

## Sprint 6: Генерация документов (Недели 12-13)

### AI Core
- [ ] Генератор заявления о банкротстве (шаблон + заполнение данными)
- [ ] Генератор реестра кредиторов
- [ ] Генератор описи имущества
- [ ] RAG: индексация ФЗ-127, АПК, ГК РФ
- [ ] Проверка документов на соответствие требованиям АПК

### Backend
- [ ] API генерации документов
- [ ] Шаблоны документов (DOCX + PDF)
- [ ] Версионирование сгенерированных документов
- [ ] Human-in-the-loop: юрист проверяет → утверждает → документ финализируется

### Frontend
- [ ] Интерфейс генерации документов (кнопка → превью → правки → утверждение)
- [ ] Сравнение версий документа

### Definition of Done
- Юрист нажимает «Сгенерировать заявление» → AI создаёт на основе данных дела
- Юрист проверяет, правит, утверждает
- Документ сохраняется в дело

### Backend
- [ ] API интеграции с kad.arbitr
- [ ] Webhook для Telegram и WhatsApp
- [ ] Автоматическое обновление статуса дела по данным из суда

### Definition of Done
- Система автоматически обнаруживает новое заседание → уведомляет юриста и клиента

```
  └── Sprint 8 (мобилка) ─── параллельно после Sprint 5
```
